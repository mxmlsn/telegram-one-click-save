**Это правило действует ВСЕГДА, в каждом чате на этом проекте.**

При любых изменениях в коде Claude **ОБЯЗАН** самостоятельно проверить результат через Chrome DevTools MCP:

1. **Перед началом работы** — подключиться к браузеру (порт `9222`), убедиться что MCP работает
2. **После любых изменений** — сделать скриншот, прочитать консоль, проверить что ничего не сломалось
3. **При работе с файлами** — загрузить тестовые файлы на сайт и проверить отображение
4. **При багах** — самостоятельно читать console logs, network requests, делать скриншоты

**НЕ проси пользователя:**
- Копировать логи консоли — читай сам через `list_console_messages`
- Делать скриншоты — делай сам через `take_screenshot`
- Описывать что видит — смотри сам через `take_snapshot`
- Загружать файлы — загружай сам из папки тестовых файлов

**Порт Chrome DevTools: `9222`** (всегда, если пользователь не сказал иное)

## Browser Debugging (Chrome DevTools MCP)

Настроен Chrome DevTools MCP для автономного дебага расширения.

### Запуск Chrome с отладкой
```bash
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --remote-debugging-port=9222 --user-data-dir=/tmp/chrome-debug
```

### Что доступно через MCP
- Скриншоты страниц и элементов
- Console messages (log/error/warn)
- Network requests
- Выполнение JS на странице
- Клики, навигация, заполнение форм
- Performance traces, memory snapshots

### Тестовые файлы
Папка с файлами для тестирования: `/Users/maximlyashenko/Desktop/new tab/filesforclaude/`
Содержит: PNG, MP4, PDF, GIF, AEP, AI, ZIP и другие форматы.

### Загрузка файлов на сайт
- Сайт: `stash.mxml.sn` — viewer.js загружается с сервера, НЕ из локального расширения
- Для загрузки файлов: инжектить `<input type="file">` через `evaluate_script`, затем `upload_file`
- Или использовать существующий drag & drop (quick save / create note)
- В локальном `web-viewer/index.html` добавлена кнопка Attach (скрепка) — при деплое будет работать нативно

### Workflow дебага
1. Пользователь запускает Chrome с `--remote-debugging-port=9222`
2. Загружает расширение через `chrome://extensions` → Load unpacked
3. Claude **сам** делает всё: навигирует, загружает файлы, смотрит скриншоты, читает консоль, находит баги
4. Пользователю НЕ нужно копировать логи — Claude видит всё через MCP
5. Пользователю нужно только: запустить Chrome и загрузить расширение

## Dev-режим для страницы настроек

Сейчас активен dev-режим для переверстки страницы настроек.

### Структура

```
dev/                    # Dev-версия для локальной разработки
  index.html           # HTML страницы настроек
  options.css          # Стили
  options.js           # JS с mock chrome.storage (данные в памяти)

options/               # Продакшн-версия (используется расширением)
  index.html
  options.css
  options.js
```

### Как работать

1. **Локальный сервер** уже запущен: http://localhost:8080/dev/index.html
   - Если сервер не работает, запусти из корня проекта:
     ```bash
     python3 -m http.server 8080
     ```

2. **Вноси правки в файлы в папке `dev/`**
   - После изменений просто обнови страницу в браузере (F5)
   - `chrome.storage` замокан - настройки сохраняются в памяти и логируются в консоль

3. **Когда верстка готова**, замени продакшн-версию:
   ```bash
   cp dev/index.html options/index.html
   cp dev/options.css options/options.css
   ```

4. **JS файл требует ручной обработки!**
   - В `dev/options.js` в начале есть mock `chrome.storage` (строки 1-60)
   - При копировании в продакшн нужно УДАЛИТЬ этот mock-блок
   - Или просто вноси изменения только в HTML/CSS, а JS копируй вручную нужные части

### Важно

- Иконки загружаются из `../icons/` - это работает и в dev, и в prod
- После замены файлов перезагрузи расширение в Chrome: `chrome://extensions` → кнопка обновления
